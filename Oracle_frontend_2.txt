Oracle Amigo Frontend Redesign Plan
1. Current State Assessment
What exists now:
- ui/src/components/StreamLikeChat.tsx — 1338-line monolith handling auth, conversations, messaging, approvals, and diagnostics all in one file
- components/ui/ — 50 shared prompt-kit/shadcn-style components that are NOT used by the main chat
- ui/src/hooks/queries.ts — TanStack Query hooks that are NOT consumed by the main chat (it uses manual useState+useEffect+setInterval polling instead)
- Design: Slack-inspired cream/aubergine light theme with custom CSS (1252 lines in styles.css)
- No client-side router — single-page SPA with no URL-based navigation
- Auth — embedded inline CloudConnectForm inside the chat view (no dedicated auth screen)
- Admin portal — separate ui-admin/ SPA with hash-based routing, well-structured
- HeroUI v3 already installed; Flowbite NOT installed
Problems to fix:
 1. Monolithic component is unmaintainable
 2. 50 shared components exist but are unused
 3. Light-purple theme feels beginner-level
 4. No dedicated auth/enrollment screens
 5. No right-inspector panel
 6. No routing for multi-screen flows
 7. No virtualization for message list
 8. No offline outbox UI
 9. No file-request detection chip
10. Approval card is basic
11. No transfer progress visualization
12. No task timeline cards
13. Admin portal is functional but could be more polished
2. Target Architecture

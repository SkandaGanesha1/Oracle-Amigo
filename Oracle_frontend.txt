You are Codex/OpenCode working inside this repository:

https://github.com/SkandaGanesha1/Oracle-Amigo

Act as the world’s top Senior Frontend Engineer and Agentic AI Product Engineer.

Your task is to completely redesign and integrate the Oracle Amigo Agentic Chat frontend and Admin UI into a next-generation agentic communication application.

The current UI looks like a beginner-level chat/dashboard. It has functional pieces, but the visual hierarchy, interaction model, component system, spacing, typography, message design, approval cards, agent workflow cards, and admin surfaces are not yet world-class.

Your goal is to transform Oracle Amigo into a premium next-generation Agentic Chat application: the next era after Slack/WhatsApp, but built for personal agents, A2A relay tasks, file-request approvals, local file search, agent-to-agent communication, auditability, and enterprise monitoring.

============================================================

Oracle Amigo currently has:

1. Local Agent
   - local single-user personal agent
   - file search
   - approval workflow
   - sqlite-vec retrieval
   - local storage
   - A2A task handling
   - cloud clients
   - heartbeat
   - relay polling
   - approval-to-transfer orchestrator

2. Control Plane
   - auth
   - enrollment
   - directory
   - contacts
   - presence
   - A2A relay
   - file transfer relay
   - audit
   - admin APIs

3. Agentic Chat UI
   - auth/enrollment
   - directory
   - conversations
   - file request detection
   - approvals
   - received files
   - audit
   - diagnostics
   - settings

4. Admin Portal
   - user/device/agent monitoring
   - presence
   - relay tasks
   - transfers
   - audit
   - revocation/security

The UI must support this product spine:

signup
→ login
→ enroll device
→ register personal agent
→ heartbeat
→ directory visibility
→ start conversation
→ send normal message
→ detect file request
→ send A2A file request

4. Mic and speech recognition design

For the visual waveform, use Web Audio API:

navigator.mediaDevices.getUserMedia({ audio: true })
AudioContext
AnalyserNode
frequency/time-domain samples
canvas/SVG bars

getUserMedia() requests user permission and returns a MediaStream; browsers must show permission/recording indicators because microphone capture is privacy-sensitive. It is available in secure contexts, including localhost, but users must grant permission. (MDN getUserMedia)

For real-time transcript, use a layered adapter:

SpeechTranscriber
├── BrowserSpeechTranscriber      // fastest POC
├── WhisperSidecarTranscriber     // better offline production path
└── MockTranscriber               // tests/dev

The Web Speech API provides asynchronous speech recognition through SpeechRecognition and supports speech recognition from microphone/audio sources, but browser/WebView behavior can vary. (MDN Web Speech API)

For production/privacy, use whisper.cpp as a local sidecar. It is a lightweight C/C++ Whisper implementation and explicitly supports building offline voice assistant applications; it also supports voice activity detection through VAD models, which is useful for cleaner push-to-talk transcripts. (whisper.cpp)

Recommended path:

Phase 1:
BrowserSpeechTranscriber + typed fallback

Phase 2:
whisper.cpp sidecar

Phase 3:
streaming partial transcript + VAD + local model selection

6. Window positioning

For correct “right side above taskbar” behavior, do not just call center().

In Rust, position the overlay near the Windows work area. The best Windows-native method is to use the work-area rectangle, which excludes the taskbar.

Implement a Tauri command:

#[tauri::command]
fn show_voice_window(app: tauri::AppHandle) -> Result<(), String> {
    let window = app.get_webview_window("voice-launcher")
        .ok_or("voice-launcher window not found")?;

    let width = 420;
    let height = 180;
    let margin_right = 24;
    let margin_bottom = 24;

    // Better: use Windows GetMonitorInfo rcWork.
    // Fallback: use primary monitor size.
    let monitor = window.primary_monitor()
        .map_err(|e| e.to_string())?
        .ok_or("no primary monitor")?;

    let pos = monitor.position();
    let size = monitor.size();

    let x = pos.x + size.width as i32 - width - margin_right;
    let y = pos.y + size.height as i32 - height - 96 - margin_bottom;

    window.set_size(tauri::Size::Physical(tauri::PhysicalSize { width, height }))
        .map_err(|e| e.to_string())?;

    window.set_position(tauri::Position::Physical(tauri::PhysicalPosition { x, y }))
        .map_err(|e| e.to_string())?;

    window.show().map_err(|e| e.to_string())?;
    window.set_focus().map_err(|e| e.to_string())?;

    Ok(())
}

Then improve later with GetMonitorInfoW and rcWork to handle taskbar position accurately.

9. White waveform component

Example SVG bars:

function WhiteWaveform({ level, active }: { level: number; active: boolean }) {
  const bars = 21;

  return (
    <svg className="waveform" viewBox="0 0 220 64" aria-hidden="true">
      {Array.from({ length: bars }).map((_, i) => {
        const center = Math.abs(i - Math.floor(bars / 2));
        const base = 12 + (bars / 2 - center) * 2.2;
        const motion = active ? Math.sin(Date.now() / 120 + i * 0.6) * 10 * level : 0;
        const h = Math.max(8, base + motion);
        const x = 10 + i * 10;
        const y = 32 - h / 2;

        return (
          <rect
            key={i}
            className="wave-bar"
            x={x}
            y={y}
            width="5"
            height={h}
            rx="2.5"
          />
        );
      })}
    </svg>
  );
}

For smoother animation, drive it through requestAnimationFrame and Web Audio AnalyserNode.
